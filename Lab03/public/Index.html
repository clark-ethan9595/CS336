Answers for the Lab3 Exercises

  • 3.1:
    • Node.js is a platform for building event-driven, server side applications, while Express.js is a framework built on top of node.js for building server side applications. Express.js provides some basic framework so that you don't have to code everything and use it's methods to ease server side development using Node.js. Express can facilitate a rapid development of Node based web applications.
    • The packages.json file contains various informational data relevant to the project. It is used to give information to npm (Node Package Manager) that allows it to identify the project as well as handle the project's dependencies.
  • 3.2:
    • These files are static in the sense that they are not being created dynamically with server-side code. Anything that happens in the browser does not affect or change the static files themselves.
    • The server does not continue to serve up the hello-world message. It only serves up the message when the url/server is called, not continuously. If the page is reloaded, then the message gets served up again.
